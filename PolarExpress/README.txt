Для реализации операций, предусмотренных данным техническим заданием, были разработаны следующие SQL-запросы:
POST: localhost:8080/tracking
{
    "moving": "Получение",
    "postOfficeId": 3,
    "postMailId": 1
}
Данный запрос выполняет операции по регистрации связи между почтовым отправлением и почтовым отделением.
На поле moving установлено ограничение на добавление значений.
Возможные варианты: Регистрация, Прибытие, Убытие, Получение.
Поля postOfficeId и postMailId выступают в роли внешних ключей, обеспечивая связь с соответствующими записями в других таблицах.


GET: http://localhost:8080/status-history/2
Запрос предназначен для извлечения статуса и полной истории перемещения почтового отправления. В процессе тестирования данной операции использован идентификатор id = 2, что соответствует предварительно загруженной информации в базе данных.
